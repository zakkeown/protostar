## Summary

Phase 10.1 is a mechanical boundary pass across 20 packages. The primary deliverables are: a `protostar.tier` field in every `package.json` (the load-bearing source of truth for the workspace-wide conformance gate), per-package no-fs/no-net contract tests on the two packages currently missing them, a single workspace-wide tier-conformance test in `admission-e2e`, manifest hygiene (uniform exports, `sideEffects`, `engines`), publish wiring for `apps/factory-cli`, and a grep-driven back-edge audit.

The most consequential findings from codebase investigation: (1) three of the four flagged "back-edges" are active cross-package imports requiring a decision, but `intent → authority` is a **phantom dep** — `@protostar/intent` declares `@protostar/authority` as a dep but zero source files import it. (2) Phase 10 locks Q-17 (public packages), Q-19 (knip), and Q-21 (authority-boundary contract) are cited as "locks 10.1 builds on" but none has landed yet — the manifest sweep will be doing Q-17 work as well. (3) The `admission-e2e` test runner uses a flat glob (`dist/*.test.js`), so the tier-conformance test must land at the top level of `src/`, not in a `contracts/` subdirectory as CONTEXT.md casually implies.

**Primary recommendation:** Start with the tier-classification plan (BOUNDARY-01/02) because every other plan's conformance assertions reference `protostar.tier`. The back-edge audit (Plan 7) must run last because its decisions may alter tier or dep declarations.

### Edge 2: `review → delivery` — TYPE-ONLY import (accept + document)

Imported symbol: `AuthorizationPayload` (interface) + `isAuthorizationPayload` (runtime validator) from `@protostar/delivery/authorization-payload` subpath export.

```typescript
// packages/review/src/delivery-authorization.ts:5
import {
  isAuthorizationPayload,
  type AuthorizationPayload,
} from "@protostar/delivery/authorization-payload";
```

**Context:** `review` uses `AuthorizationPayload` to validate a stored payload during `reAuthorizeFromPayload`. The type describes the persisted shape of a payload that `review` itself mints as `DeliveryAuthorization`. The `delivery` package owns the payload schema; `review` owns the brand.

**tsconfig drift note:** `packages/review/tsconfig.json` is missing `@protostar/delivery` in its refs despite having it as a dep. This is an existing TS project-graph bug.

**Recommended action: Accept the dep + document in AGENTS.md forbidden-edge prose.** The `delivery/authorization-payload` subpath exists specifically for this cross-package type bridge. Extracting `AuthorizationPayload` to a neutral package would require creating a types-only package — over-engineering for one interface. Accept and document: "review → delivery is permitted for authorization payload type-bridge only."

[VERIFIED: packages/review/src/delivery-authorization.ts lines 1–10]

### Edge 3: `review → repair` — RUNTIME import (accept + document OR flip into review)

Imported symbols: `synthesizeRepairPlan`, `computeRepairSubgraph` (both runtime values, not types) from `@protostar/repair`.

```typescript
// packages/review/src/run-review-repair-loop.ts:1
import { synthesizeRepairPlan, computeRepairSubgraph } from "@protostar/repair";
```

Called at lines 307 and 311 inside `runReviewRepairLoop`.

**Context:** `runReviewRepairLoop` is the loop body — it calls repair synthesis directly. `@protostar/repair` was created as a Phase 5 Plan 01 skeleton specifically to hold the repair-synthesis logic. The current shape: repair owns the synthesis transforms, review owns the loop that calls them.

**Two viable options:**
- **Accept + document:** The review-repair loop is the only caller; the dep direction is asymmetric but intentional. Add AGENTS.md prose: "review → repair is the sanctioned call site for repair synthesis; repair must not depend on review."
- **Flip into review:** Move `synthesizeRepairPlan` and `computeRepairSubgraph` into `packages/review/src/`. This is a source move — strictly out of 10.1 scope per Q-02.

**Recommended action: Accept + document** (flip is out of scope; the current structure is defensible since repair also has other consumers in principle).

[VERIFIED: packages/review/src/run-review-repair-loop.ts]

### Edge 4: `delivery-runtime → review` — RUNTIME import (accept + document)

Imported symbols:
- `mintDeliveryAuthorization` (function, runtime value) — in 3 test files and `execute-delivery.ts`
- `DeliveryAuthorization` (interface type) — in `execute-delivery.ts`

```typescript
// packages/delivery-runtime/src/execute-delivery.ts:3
import type { DeliveryAuthorization } from "@protostar/review";
// packages/delivery-runtime/src/execute-delivery.ts:7 (via test files)
```

**Context:** `DeliveryAuthorization` is a privately-branded type (`unique symbol`) minted exclusively by `runReviewRepairLoop`. `delivery-runtime` needs the brand to enforce that only authorized runs can execute delivery. `mintDeliveryAuthorization` appears in tests only to construct test fixtures.

**Recommended action: Accept + document.** The brand lives in `review` by design (it is the authorization issuer). Moving the brand to a neutral types package would require creating a new package with no other purpose. The prod import is type-only; only tests use the mint function. Document in AGENTS.md: "delivery-runtime → review is permitted for DeliveryAuthorization brand consumption; review is the sole issuer of this brand."

## Existing Contract Test Templates (Research Task D — VERIFIED)

### Per-Package `no-fs.contract.test.ts` Template

Lives at `packages/<pkg>/src/no-fs.contract.test.ts`. Canonical shape from `packages/delivery-runtime/src/no-fs.contract.test.ts`:

1. Import `readdir`, `readFile` from `node:fs/promises` for its own walking
2. Declare `SELF_BASENAME` + `KNOWN_CONTRACT_TESTS` set to exclude self (and sibling contracts)
3. Resolve `srcRoot` from `import.meta.url`
4. `FORBIDDEN_PATTERNS` array: `node:fs`, `node:fs/promises`, `fs`, `node:path`, `path`
5. `async function* walkTypeScriptFiles(dir)` recursive, skips `.d.ts`
6. `stripComments(source)` removes block + line comments before pattern test
7. Single `describe/it` asserting `offenders === []`

**Reuse for `lmstudio-adapter`:** copy exactly, adjust `KNOWN_CONTRACT_TESTS` to just `['no-fs.contract.test.ts']`.

### Tier-Conformance Test Logic (Q-11/Q-12/Q-13)

Per CONTEXT.md Q-12 decision: custom test, no dependency-cruiser. Algorithm:

### Cross-Package Conformance Test in admission-e2e (Pattern 2)

**Key constraint:** File MUST be at flat top of `packages/admission-e2e/src/` — NOT in a subdirectory. Test runner glob is `dist/*.test.js` (flat).

```
packages/admission-e2e/src/tier-conformance.contract.test.ts
```

Logic: glob packages → parse package.json (tier + deps) → parse tsconfig.json (refs) → assert tier rules + dep-direction rules + refs-match-deps + sideEffects + engines.

**No new dep.** Use `node:fs/promises` readFile, `node:path`, JSON.parse. Read `pnpm-workspace.yaml` as plain text (split on newlines, extract paths). No YAML parser.
