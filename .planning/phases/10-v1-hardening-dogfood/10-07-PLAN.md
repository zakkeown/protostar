       ```markdown
       Protostar is a dark software factory control plane that mutates real GitHub repositories.
       This document describes the v0.1 trust posture and how to report vulnerabilities.

       ## Trust assumptions

       - The factory operator authors and confirms intent. After confirmation, autonomy is
         governed by capability envelopes, policy, and stop gates documented in
         `packages/policy/`.
       - Filesystem authority lives only in `apps/factory-cli` and `packages/repo`
         (see [AGENTS.md](./AGENTS.md)).
       - Network authority lives only in `packages/dogpile-adapter` (LM Studio coordination)
         and `packages/delivery-runtime` (Octokit + git push).
       - All other domain packages are pure (no fs, no network).

       ## Capability envelope

       Every confirmed intent carries a typed `capabilityEnvelope` (see
       `packages/intent/src/capability-envelope.ts`) bounding repo scopes, tool permissions,
       write budgets, and repair-loop counts. Authority decisions intersect the envelope
       with policy via `packages/authority/`'s `PrecedenceDecision` brand.

       ## Secret handling

       - GitHub PAT is read once from the environment at delivery time and never logged.
         Phase 7 added redaction sanitizer; see `07-SECURITY.md`.
       - LM Studio is local-only; no cloud-LLM credentials in v0.1.
       - The dogfood-only fine-grained PAT (`protostar-dogfood`) is scoped to a single repo.

       ## Reporting

       Email zak.keown@outlook.com with subject `[protostar-security]`. Please do not open
       public issues for unpatched vulnerabilities.

       ## Supported versions

       The most recent `0.1.x` release on npm is the supported line. Prior versions are not patched.
       ```
