<objective>
Implement the dogfood driver as **two artifacts in coordination**:

1. **Internal CLI subcommand** `apps/factory-cli/src/commands/__dogfood-step.ts` — owns ALL filesystem writes (cursor, log.jsonl, report.json), seed selection, runId discovery, and CI watch. 2. **Bash orchestration script** `scripts/dogfood.sh` (Q-09 shell, dark, resumable, sequential) — a thin loop that calls `protostar-factory __dogfood-step` and the public `protostar-factory run` subcommand. Bash performs NO fs writes and NO business logic; it is the orchestration sequencer.

Persist cursor + final report.json (Q-11/Q-12), lock the report.json schema with an admission-e2e contract, then run ≥10 sequential factory invocations against the toy repo and confirm ≥80% reach `pr-ready` per Q-10's pass criterion. Justify the Phase 8 convergence-threshold tuning in writing.

Purpose: DOG-04 — the v0.1 exit gate. This plan cannot start until 10-01..10-07 commit (the driver imports the seed library (10-02), targets the toy repo (10-01), uses Phase 9 prune scope extension (10-02 Task 2), publishes after the public surface is locked, and is the literal "did the hardened factory ship" test).

**Why the rewrite from inline-bash-fs:** CONTEXT.md `<carried_forward>` and AGENTS.md lock filesystem authority to `apps/factory-cli` and `packages/repo` ONLY. The fix: move ALL of that into the subcommand under factory-cli (the fs-authorized package). Q-09's "no public dogfood subcommand" constraint is honored by the `__` prefix — the subcommand exists internally but is NOT part of the locked public CLI surface.

       ```bash
       #!/usr/bin/env bash
       # Dark dogfood driver. No fs writes; no business logic. Just orchestrates subcommand calls.
       set -euo pipefail

       RUNS=10
       SESSION_ID=""
       while [[ $# -gt 0 ]]; do
         case "$1" in
           --runs) RUNS="$2"; shift 2 ;;
           --resume) SESSION_ID="$2"; shift 2 ;;
           *) echo "unknown arg: $1" 1&gt;&amp;2; exit 2 ;;
         esac
       done

       if [[ -z "$SESSION_ID" ]]; then
         SESSION_ID="$(date -u +%Y%m%dT%H%M%SZ)-$RANDOM"
       fi

       CLI="node apps/factory-cli/dist/main.js"

       # Begin (idempotent — resumes if cursor exists)
       $CLI __dogfood-step --session "$SESSION_ID" --action begin --total "$RUNS"

       # Loop: ask subcommand for next seed; if no more seeds, the subcommand exits non-zero with
       # "session complete" on stderr — we treat that as the terminal condition.
       while true; do
         set +e
         SEED_JSON=$($CLI __dogfood-step --session "$SESSION_ID" --action next-seed --json 2&gt;/dev/null)
         RC=$?
         set -e
         if [[ "$RC" -ne 0 ]]; then break; fi

         SEED_ID=$(echo "$SEED_JSON" | jq -r .seedId)
         SEED_INTENT=$(echo "$SEED_JSON" | jq -r .intent)
         RUN_START="$(date -u +%Y-%m-%dT%H:%M:%SZ)"

         # Snapshot existing runs BEFORE invocation (subcommand owns the path)
         BEFORE_SNAPSHOT=$(mktemp)
         $CLI __dogfood-step --session "$SESSION_ID" --action snapshot-runs --out "$BEFORE_SNAPSHOT"

         # Invoke the public `run` subcommand. Phase 9 surface; no fs writes from bash.
         set +e
         GITHUB_TOKEN="$PROTOSTAR_DOGFOOD_PAT" \
           pnpm run factory -- run \
             --intent "$SEED_INTENT" \
             --target-repo "https://github.com/zkeown/protostar-toy-ttt.git" \
             --archetype cosmetic-tweak \
             1&gt;/dev/null 2&gt;/dev/null
         set -e

         # Discover the new runId via subcommand (computes diff against snapshot)
         RUN_ID=$($CLI __dogfood-step --session "$SESSION_ID" --action discover-run-id --before-snapshot "$BEFORE_SNAPSHOT" 2&gt;/dev/null || echo "")

         # Read prUrl from the run's manifest (subcommand exposes this; fall back to empty)
         PR_URL=""
         if [[ -n "$RUN_ID" ]]; then
           PR_URL=$($CLI inspect --run-id "$RUN_ID" --json 2&gt;/dev/null | jq -r '.delivery.prUrl // .prUrl // empty' || echo "")
         fi

         OUTCOME="no-pr"
         CI_VERDICT=""
         if [[ -n "$PR_URL" ]]; then
           # Subcommand watches CI and returns verdict as JSON
           CI_JSON=$($CLI __dogfood-step --session "$SESSION_ID" --action watch-ci --pr-url "$PR_URL" --timeout-seconds 600 --json)
           CI_VERDICT=$(echo "$CI_JSON" | jq -r .verdict)
           case "$CI_VERDICT" in
             success) OUTCOME="pr-ready" ;;
             timeout) OUTCOME="ci-timeout" ;;
             failure) OUTCOME="ci-failed" ;;
           esac
         fi
         RUN_END="$(date -u +%Y-%m-%dT%H:%M:%SZ)"

         # Record (subcommand handles cursor R-M-W + log.jsonl append atomically)
         $CLI __dogfood-step --session "$SESSION_ID" --action record \
           --runId "$RUN_ID" \
           --pr-url "$PR_URL" \
           --ci-verdict "$CI_VERDICT" \
           --outcome "$OUTCOME" \
           --started-at "$RUN_START" \
           --finished-at "$RUN_END"

         rm -f "$BEFORE_SNAPSHOT"
       done

       # Finalize: subcommand writes report.json + emits the single stderr summary
       $CLI __dogfood-step --session "$SESSION_ID" --action finalize
       ```
       Make executable: `chmod +x scripts/dogfood.sh`.

       Notes on this script:
       - Bash performs ZERO fs writes to `.protostar/dogfood/`. Every state change goes through `__dogfood-step`.
       - `mktemp` for `BEFORE_SNAPSHOT` is allowed (system tmp; not under `.protostar/`); it's a transient handle the subcommand reads.
       - `jq` reads on stdout from subcommands is NOT business logic — it's parsing the subcommand's `--json` contract.
       - The single stderr summary is emitted by `finalize`, not by bash.
