<objective>
Two narrow CLI extensions to make replay bundles first-class operator-surface citizens:
1. **inspect overlay-aware** (REPLAY-09): resolves `overlay.json` pointers to parent-bundle artifacts, verifies `originalSha256` on every overlay-redirected read, refuses on mismatch (Pitfall #4 — overlay tampering).
2. **prune carve-out** (REPLAY-10): `replays/` directories are never auto-pruned in v1 per D-09, regardless of `--older-than` horizon.

Both changes are additive; no behavior change for runs without replays.

Purpose: replay bundles must be inspectable (operators need post-mortem visibility) and protected from accidental prune. Without this plan, replays exist but are operationally dark.
Output: inspect/prune updated + help snapshots refreshed.
</objective>
