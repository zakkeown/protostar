<objective>
Two pure-tier helpers that make replay possible:
1. **Dependency-closure resolver** (D-05): given an edited path set and the recorded `pathsRead.json` records from a parent run, returns the ordered set of stages that must be re-executed (cascaded) and the set that can be served from overlay (preserved).
2. **Per-stage hash-chain harness** (D-06): asserts byte-equality of every `(stageInstance, inputsHash, outputsHash)` against a recorded fixture; on mismatch, reports the *first* stage that diverged, not just bundle-level drift.

Both are pure functions, fully unit-tested, and produce structured output the CLI/regression-fixture can act on.

Purpose: replay's CLI (plan 13-07) calls `resolveCascade` to know which stages to re-execute; the regression fixture (plan 13-11) calls `assertHashChain` to verify determinism. Without these, plans 13-07 and 13-11 cannot exist.
Output: pure-tier resolver + harness exported from `@protostar/replay`.
</objective>
