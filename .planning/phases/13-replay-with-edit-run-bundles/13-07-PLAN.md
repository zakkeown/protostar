<objective>
Land the `factory-cli replay` verb end-to-end. This plan composes all prior plans (13-02 schemas, 13-03/04/05 sidecars, 13-06 resolver + hash chain) into a working CLI that:
1. Parses operator inputs (`--run`, `--edit`, `--edit-set`, `--replay-id`, `--allow-delivery`).
2. Validates the edit (allowlist, schema-validate-on-entry, single-edit ceiling).
3. Loads the parent run's `pathsRead.json` sidecars and computes the cascade.
4. Re-executes only cascaded stages (calling existing producer entry points with edited inputs).
5. Writes the sibling replay bundle (`replays/<replayId>/`) with overlay manifest pinning preserved artifacts by hash.
6. Extends `manifest.json` with replay-specific fields per REPLAY-17.

Note: this plan does NOT cover delivery (plan 13-09), inspect/prune awareness (plan 13-08), or counterfactual evaluation (plan 13-10). Those compose on top.

Purpose: this is the integration point where pure-tier primitives meet fs-tier authority. Without this plan, no replay can run.
Output: working `protostar-factory replay` command + sibling bundle layout.
</objective>
